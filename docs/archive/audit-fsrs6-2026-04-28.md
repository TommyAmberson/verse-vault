### Soundness concerns (verse-vault extension)

**S1. Linear-blend update is not a defensible generalization of FSRS-6.** `apply_weighted_update`
evaluates `next_states` once at the _current_ state, then does
`s_delta += w_i Ă— (target_i âˆ’ current)`. FSRS's S-update is highly nonlinear. Mixing
`(Again, 0.5) + (Good, 0.5)` can cancel to â‰ˆzero net change even though both observations are
individually informative. Sequential application gives a different result. Consider resolving to a
single grade and running one FSRS step, or applying sequentially.

**S2. Per-edge total weight is unbounded across hidden atoms.** `credit::assign_credit` normalizes
weights _within_ a single hidden atom's surviving paths. An edge participating in paths for multiple
hidden atoms accumulates separate entries with no global cap (`credit.rs:206-216`). For a 4-phrase
verse, an edge could receive total weight â‰ˆ4.0, overshooting the FSRS target by 4Ă—. The docstring
(`fsrs_bridge.rs:102-105`) says weight=1.0 is a full update â€” implying weight âˆˆ [0, 1] was intended.

**S3. `last_review_secs` is unconditionally reset to `now_secs` regardless of weight**
(`fsrs_bridge.rs:131`). A Î²=0.2 exposure update fully refreshes the timestamp. Future R(t) starts
from near-zero elapsed days. Possibly threshold the timestamp update on total weight.

**S4. `days_elapsed as u32` truncates sub-day intervals to 0** (`fsrs_bridge.rs:94`). A review at
23h after the last â†’ `delta_t = 0` â†’ routes to `stability_short_term`. A review at 24h+1s â†’
`delta_t = 1` â†’ normal path. Discrete jump at 24h not in spec; matters for same-session re-drills,
which is verse-vault's exact use case.

## Layer 2 â€” Credit assignment (`credit.rs`)

### Bugs

**B3. `assign_blame_for_fail` ignores `decay_multiplier`** (`credit.rs:219-260`). Credit (line 195)
multiplies path probability by `ap.decay_multiplier`. Blame (line 235-249) does not. A weak anchor
path five verses away (multiplier â‰ˆ 0.77) gets the same blame attribution as a direct path with the
same weakest edge. Asymmetric, undocumented in `docs/review.md`.

**B4. Missing grade silently treated as Good** (`credit.rs:78-79`):

```rust
let grade = review.grades.get(&hidden_atom).copied().unwrap_or(Grade::Good);
```

If a caller forgets to grade a hidden atom, blame disappears and phantom credit lands on every path.
`engine.rs:62-72` does no validation that `grades` covers `card.hidden`.

### Soundness concerns

**S5. Structural-vs-learnable edge distinction is in `docs/review.md` but not in code.** Spec at
`docs/review.md:101-102`: _"Structural edges (no FSRS state) contribute R = 1.0 but cannot receive
credit or blame."_ But `Edge` always carries an `EdgeState`. R=1.0 holds accidentally because
`retrievability` early-returns 1.0 for `stability <= 0.0`. The "cannot receive credit or blame" half
is not enforced â€” a Good update will move S from S_MIN to the FSRS target, "promoting" a structural
edge.

**S6. `apply_reverse_reinforcement` order-dependence** (`credit.rs:295-331`). HashMap iteration
order picks which primary's echo lands on a shared reverse candidate. Rare in verse graphs; iterate
sorted for determinism.

### Spec parity (matches docs)

Source set construction, `failed_atoms` definition, hop limit, anchor transfer scoping (VerseRef +
same chapter only), credit weighting, weakest-edge blame, fallback chain priority all match
`docs/review.md`. The `fallback_chain_no_double_counting` test exercises the invariant correctly.

## Layer 3 â€” Review pipeline (`engine.rs`)
